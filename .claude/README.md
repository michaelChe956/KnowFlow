# Claude 文档管理中心

## 目录概述
此目录是 KnowFlow 项目中所有 Claude 生成文档的统一存储和管理中心。

## 目录结构

```
.claude/
├── docs/           # 需求文档
├── designs/        # 方案设计
├── readmes/        # README 文档
├── architecture/   # 系统架构
├── notes/          # 开发笔记
├── analysis/       # 分析报告
└── logs/           # 开发日志
```

## 各目录用途

### 📁 docs/ - 需求文档
- 功能需求说明
- 用户需求分析
- 业务需求文档

### 📁 designs/ - 方案设计
- 架构设计方案
- 技术方案文档
- 实现方案设计
- 接口设计文档

### 📁 readmes/ - README 文档
- 项目说明文档
- 模块使用指南
- 部署说明文档

### 📁 architecture/ - 系统架构
- 系统架构分析
- 技术架构文档
- 微服务架构设计

### 📁 notes/ - 开发笔记
- 开发过程记录
- 临时技术笔记
- 问题解决记录

### 📁 analysis/ - 分析报告
- 代码分析报告
- 技术调研报告
- 性能分析文档

### 📁 logs/ - 开发日志
- 开发日志记录
- 问题追踪文档
- 变更记录

## 使用规则

### 强制要求
- ✅ **必须**将所有 Claude 生成的文档放入对应的子目录
- ❌ **禁止**将文档随意放置在项目根目录或其他位置
- 📝 **必须**使用中文编写所有文档内容
- 🗂️ **建议**定期整理和归档文档

### 命名规范
- 使用中文命名，清晰表达文档内容
- 格式：`文档名称_版本号_日期.md`
- 示例：`用户认证系统设计方案_v1.0_20241205.md`

### 维护原则
- 保持目录结构清晰有序
- 定期清理过期或无用的文档
- 确保文档内容的准确性和时效性
- 便于团队成员快速查找和维护

## 文档流程

1. **创建**：根据文档类型选择对应目录
2. **编写**：按照各目录的规范编写内容
3. **命名**：使用统一的命名规范
4. **维护**：定期更新和整理文档
5. **归档**：对过期文档进行适当处理

---

**重要提醒**：此目录结构是 Claude 文档管理的强制规则，请严格遵守以确保项目文档的统一性和可维护性。
